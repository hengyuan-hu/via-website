Your coding/computer-use agent is secretly a robot control agent.

Abstract

Core Idea

  • Recast robot control as a visual tool-use task for agents, similar to general computer use.
  • Design a visual interface and a small set of versatile MCP tools for agents to see and command the robot.
  • Run off-the-shelf agents (Claude Code or Codex) to drive the interface and complete robot manipulation tasks.

Result Highlights

  • VIA solves a diverse suite of tabletop manipulation tasks zero-shot with a high success rate.
  • VIA inherits strong reasoning, long-horizon planning, and closed-loop control capabilities from frontier agents for free.
  • VIA rides the wave of foundation model scaling, with VIA (Fable) significantly outperforming VIA (Opus).

Note

  • VIA is not Code-as-Policies (CaP). It involves no CaP-style code generation and provides no perception or skill primitives for the model to compose. VIA observes the UI through raw pixels (screenshots) and acts via low-level tools akin to mouse and keyboard operations in computer-use.
  • VIA opens up a new opportunity for leveraging state-of-the-art foundation models at scale in robotics, and demonstrates that with the right interface, robot control can become another economically valuable agentic task that benefits directly from modern models and agents.

VIA

Evaluation

We use a suite of six tasks drawn from different benchmarks, together covering diverse manipulations.

We evaluate VIA using two families of agents:

  • Claude Code with Opus 4.8 (CC-Opus) and Fable 5 (CC-Fable),
  • Codex with GPT-5.5 (Codex-5.5) and GPT-5.6-Sol (Codex-5.6-Sol).

The reasoning effort is set to xhigh for all agents.

On top of a shared system prompt on how to use the interface, we consider two variants of task-specific prompts for each task:

  • minimal: states only the task's goal and success condition, testing the zero-shot performance of each agent.
  • detailed: provides an example list of waypoints, simulating the effect of having a demonstration.

We disable memory read/write for the agents so that they cannot gain cross-episode experience.
